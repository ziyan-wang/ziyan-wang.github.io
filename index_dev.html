Short Biography

Ziyan is a full stack developer and a researcher. He values practice no less than theory.

Ziyan devotes himself to making programming languages and tools better. Specifically, he wants to simplify the way people learn programming and the way developers work, by designing safe general-purpose and domain-specific languages with high development productivity and diverse programming paradigms.

Why? It's Ziyan's observation that a large proportion of software defects, i.e. bugs and vulnerabilities, though seemingly nothing more than errors in code, can be ultimately dated back to the design flaws of languages. If there are better languages, such issues can be eliminated at the very root, far before developers start coding.

However, Ziyan understands that proposing novel languages may be too ambitious, running the risk of not seeing real usage. He thus makes a compromise to also adopt program analysis as a practical way to improve code written in existing flawed languages.
